## Executive Summary

The TradSL system has **22 confirmed bugs**, of which **4 are CRITICAL**. The demo appears to work (trades execute) but the strategy is **NOT actually using**:
- Computed features (EMA, returns, correlations)
- The trainable model from the DSL
- The configured test period

Instead it just uses raw close prices with a simple RL agent.

## Critical Bugs

### Bug #1: test_start Ignored - Hardcoded 70% Split

**Location:** `tradsl/__init__.py`, test() function, line ~635

**Description:**
The `_backtest` configuration accepts a `test_start` parameter (e.g., '2024-07-01') which should define when the test period begins. However, this value is completely ignored.

**Current Code:**
```python
# In test() function
test_start_idx = int(len(data) * 0.7)  # Hardcoded 70%!
```

**Problem:**
- User specifies `test_start: 2024-07-01` in DSL
- The string is read but never converted to an index
- Instead, a hardcoded 70% split is always used

**Solution:**
```python
# Convert test_start string to actual index
test_start_config = backtest_config.get('test_start')
if test_start_config and data is not None:
    try:
        import pandas as pd
        test_date = pd.Timestamp(test_start_config, tz='UTC')
        mask = dates >= test_date
        if mask.any():
            test_start_idx = int(np.argmax(mask))
        else:
            test_start_idx = int(len(data) * 0.7)
    except:
        test_start_idx = int(len(data) * 0.7)
else:
    test_start_idx = int(len(data) * 0.7)
```

### Bug #2: DAG Features Never Computed

**Location:** `tradsl/__init__.py`, _load_data_from_adapters() and execute_bar_fn()

**Description:**
The DSL defines features like returns, EMA, correlations, but these are **never actually computed**. Only raw close prices are used.

**Solution:**
Pre-compute all function-node features before training:

```python
from tradsl.features import FUNCTION_REGISTRY
from tradsl.circular_buffer import CircularBuffer

def _compute_dag_features(resolved, dag, raw_data):
    n_bars = min(len(v) for v in raw_data.values())
    node_arrays = {}
    
    for name in dag.metadata.source_nodes:
        if name in raw_data:
            node_arrays[name] = raw_data[name][:n_bars]
    
    for node_name in dag.metadata.execution_order:
        block = resolved.get(node_name, {})
        func_name = block.get('function')
        if func_name is None:
            continue
        
        spec = FUNCTION_REGISTRY.get(func_name)
        if spec is None:
            continue
        
        inputs = block.get('inputs', [])
        params_ref = block.get('params')
        params = resolved.get(params_ref, {}) if params_ref else {}
        
        min_lookback = spec.min_lookback
        if 'period' in params:
            min_lookback = params['period']
        buf_size = min_lookback
        
        result = np.full(n_bars, np.nan)
        
        if len(inputs) == 1 and inputs[0] in node_arrays:
            arr = node_arrays[inputs[0]]
            buf = CircularBuffer(buf_size)
            for i in range(n_bars):
                buf.push(arr[i])
                data_slice = buf.to_array()
                if data_slice is not None:
                    val = spec.func(data_slice, **params)
                    result[i] = val if val is not None else np.nan
        
        node_arrays[node_name] = result
    
    feature_cols = []
    for name in dag.metadata.execution_order:
        if name in node_arrays and name not in dag.metadata.source_nodes:
            feature_cols.append(node_arrays[name])
    
    return np.column_stack(feature_cols) if feature_cols else None
```

### Bug #3: Trainable Model Disconnected from Agent

**Location:** `tradsl/__init__.py`, train() - separate model creation

**Description:**
Two separate ML systems exist but are NOT connected:
- Agent Policy: trained via RL (experiences + rewards)
- Trainable Models: trained via supervised learning (features + labels)

**Solution:**
Append trainable model's prediction as an additional feature:

```python
def execute_bar_fn(bar_idx, bar_data, portfolio=None):
    observation = bar_data.copy()
    
    extra_signals = []
    for name, model_info in trainable_models.items():
        model = model_info.get('model')
        if model is not None and model.is_fitted:
            pred = model.predict(bar_data)
            extra_signals.append(pred)
        else:
            extra_signals.append(0.0)
    
    if extra_signals:
        observation = np.append(observation, extra_signals)
    
    action, conviction = agent.observe(observation, portfolio_state, bar_idx)
```

### Bug #4: Data Loaded Twice

**Location:** `tradsl/__init__.py`, run() function

**Description:**
When `run()` is called without providing `data`, both `train()` and `test()` load data separately.

**Solution:**
Load data once and pass through:

```python
def run(dsl_string, context=None, data=None, checkpoint_dir='/tmp/tradsl_checkpoints'):
    raw = parse(dsl_string)
    validated = validate(raw)
    resolved = resolve(validated, context)
    backtest_config = resolved.get('_backtest', {})
    
    dates = None
    if data is None:
        result = _load_data_from_adapters(resolved, backtest_config)
        if result is None:
            raise ValueError('Could not load data from adapters.')
        data, dates = result
    
    train_result = train(dsl_string, context, data, checkpoint_dir)
    test_result = test(dsl_string, train_result.checkpoint_path, data, context, dates)
```
